Preguntas

  • ¿Cuál es la diferencia entre Internet y la World Wide Web?

    • El internet es una "red de redes", es descentralizado y conecta millones de computadoras en todo el mundo. La WWW es un modelo de distribución de información que utiliza el protocolo http y que puede accederse por medio de internet. [1]

  • ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

    • GET: Recuperar cualquier información en forma de entidad; recupera un recurso.
    HEAD: Recupera líneas y headers pero no el body.
    POST: Enviar información al servidor; crea un recurso.
    PUT: Actualiza un recurso existente, no lo crea si no existe.
    PATCH: Realizar la actualización parcial de un recurso, a diferencia de PUT no suprime las propiedades de recurso local que no están incluidas en la petición.
    DELETE: Suprime un recurso. [2] [3]

  • ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

    • POST. Porque se está enviando información y es más seguro.

  • ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

    • GET.

  • Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

    • La solicitud ha tenido éxito. El significado de un éxito varía dependiendo del método HTTP. [4]

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

    • Error 404: Cuando un servidor web recibe una solicitud de una URL que no puede localizar, devuelve un código de estado HTTP de error 404 Not Found. Este código de error significa que el navegador fue capaz de resolver el número de IP asignado a un dominio y encontrar el servidor correctamente, pero que éste no fue capaz de encontrar el fichero solicitado. [5] A veces sí es responsabilidad del desarrollador corregir el error ya que se trata de un defecto en la página o se eliminó algo incorrectamente. Sin embargo también puede tratarse de un error del usuario si ingresó mal la URL.

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

    • Error 500: Significa que algo está mal en el servidor del sitio, pero el servidor no pudo especificar cuál es el problema exacto. [6] Sí es responsabilidad del desarrollador porque significa que hay problemas con el código o con los permisos de acceso.

  • ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

    • Significa que su uso ya no es recomendado porque el estándar ha sido actualizado. [7] Algunos elementos desaprobados son: <br/> <div> <b>

  • ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

    • Codificación de caracteres como el DOCTYPE; HTML5 define de forma más estricta los contenidos; nuevos elementos: article, aside, dialog; se cambiaron y eliminaron elementos; nuevos atributos<; cambios en sintaxis. [8]

  • ¿Qué componentes de estructura y estilo tiene una tabla?

    • Estructura: <tr>define cada fila. <th>es un encabezado de la tabla. <th>es un casillero (columna). Estilo: anchura, alineación del texto, bordes, relleno, fondo.

  • ¿Cuáles son los principales controles de una forma HTML5?

    • <label> <input> <button> [9]

Conceptos

URL: Uniform Resource Locator. Es una dirección única de las páginas de internet. Se compone por el identificador de protocolo (ej. http) y por el nombre del recurso (ejemplo.com), o sea, la dirección del recurso. [11]

MIME: Multipurpose Internet Mail Extensions. Es una forma estandarizada de indicar el formato de un documento. Los navegadores utilizan el MIME para determinar cómo se procesará el documento. Ejemplos: text, image, audio, video, application. [12]
